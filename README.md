## Project Overview

This project applies supervised machine learning to predict whether a traffic collision results in injury, using real collision records from the City of Waterloo (2015–2022). Given that a collision occurred, the goal is to classify the outcome as injury (1) or no injury (0) based on features available at the time of the crash.

**Problem Statement :** Given that a collision occurred, what factors increase the probability that the outcome results in injury rather than no injury?

**Goals:**
- Extract insights from collision data to identify the strongest risk factors for injury outcomes
- Build a binary classifier that could realistically support emergency response and road-safety decision-making
- Benchmark three ML algorithms (logistic regression, decision tree, random forest) and optimize for recall given significant class imbalance (~14% injury rate)

## Dataset

**Source:** City of Waterloo Open Data Portal  
**URL:** https://opendata.waterloo.ca  
**Years covered:** 2015–2022  
**Raw size:** 8,928 rows × 42 columns  
**Cleaned size:** 8,892 rows × 20 columns  

The dataset contains police-reported traffic collision records including location, time, environmental conditions, lighting, traffic control type, and indicators of whether pedestrians, cyclists, or motorcyclists were involved.

**Target variable:** `injury` (binary integer)
- 1 = fatal or non-fatal injury
- 0 = property damage only or non-reportable
- 9 rows with ambiguous labels ("Other", "Unknown") were removed

**Class distribution:** ~14% injury (1), ~86% no injury (0). There is a significant class imbalance.

## Methodology Summary

### Data cleaning (Notebook 01)
- Dropped 22 columns: leakage columns (CLASSIFICATIONOFACCIDENT, COLLISIONTYPE), coordinate columns, admin metadata, and redundant fields
- Filled categorical NaN values with 'Unknown'
- Removed 27 duplicate rows
- Compared LIGHT and LIGHTFORREPORT via cross-tabulation (99%+ agreement); retained LIGHT for interpretability
- Created binary injury target from CLASSIFICATIONOFACCIDENT, then dropped source column to prevent leakage

### Feature engineering & preprocessing (Notebook 02)
- Created VULNERABLE_USER_INVOLVED: combined PEDESTRIANINVOLVED, CYCLISTINVOLVED, MOTORCYCLISTINVOLVED into a single binary indicator, motivated by EDA showing injury rates of 0.61–0.95 for these groups vs. 0.14 dataset average
- Stratified 60/20/20 train/validation/test split (random_state=42)
- OneHotEncoder on 13 categorical features (handle_unknown='ignore')
- StandardScaler on 4 numerical features
- Preprocessor fit on training data only, then applied to validation and test — prevents leakage of distributional information across splits
- Final processed feature count: 16 input features → 89 features after one-hot encoding

### Models (Notebooks 03–04)
All models used class_weight='balanced' to address the 14% injury class imbalance. Primary evaluation metrics: ROC-AUC, recall, F1-score. Accuracy is reported but not used as the primary metric due to class imbalance.

**Logistic Regression (Notebook 03)**
- Baseline: no class weighting, C=1.0
- Tuning: grid search over C ∈ {0.01, 0.1, 1.0, 10.0} × class_weight ∈ {None, 'balanced'}
- Final model: C=1.0, class_weight='balanced', retrained on train+val combined before test evaluation

**Decision Tree (Notebook 04)**
- Tuning: max_depth swept over {3, 4, 5, 6, 7, 8}, selected by validation ROC-AUC
- class_weight='balanced', criterion='gini'
- Evaluated across 3 feature sets: Core, Forward-Selected, All Features

**Random Forest (Notebook 04)**
- Tuning: max_depth swept over {5, 6, 7, 8, 10, 15}, n_estimators=100
- class_weight='balanced'
- Same 3 feature sets as Decision Tree
- Forward feature selection: greedy group-level addition using validation ROC-AUC

## Results Summary

| Model | Feature set | Test ROC-AUC | Injury recall | Injury F1 |
|-------|-------------|--------------|---------------|-----------|
| Logistic Regression (baseline) | All | 0.828 (val) | 0.31 | 0.45 |
| Logistic Regression (tuned) | All | 0.812 | 0.74 | 0.40 |
| Decision Tree | All Features | 0.779 | 0.55 | 0.41 |
| Random Forest | All Features | 0.822 | 0.55 | 0.45 |

**Best recall:** Logistic Regression (tuned) — 0.74. Prioritized for safety-focused context where missing an injury is more costly than a false alarm.  
**Best ROC-AUC:** Random Forest (All Features) — 0.822.
